terminal reference sheet

i only ever use bash on my terminals because using powershell makes me want to kill myself. i use git bash for windows alongside bash in WSL ubuntu.
i'm new to all this so i've explained some things from how i understand them. feel free to e-mail me if i got something wrong though

overall

  • clear - clears the terminal screen.
  • touch [filename] - creates a file.
  • cd [dirname] - moves where you are. cd - gets you to the directory you were previously at.
  • mkdir [dirname] - makes a directory / folder.
  • exit - exit terminal.
  • history - gives you a history of your inputs.
  • help - more info.
  • start [filename] - opens a file.
  • mv [1] [2] - for renaming / moving files
  • pwd - shows you where you currently are
  • which [programname] - tells you the location path the terminal is using to locate the program.
  • ls - lists files of the directory you're currently in. ls -l for a more detailed listing. ls -a to show hidden files. ls --help help!!!
  • rm - remove a file.
  • echo - outputs whatever you give it.
  • cat [filename] - outputs the contents of that file in the terminal
  • cp [filename1] [newfile] - copies a files. you can also achieve the same with cat <[filename1]> [newfile]
  • find -name '*[search]*' - searches for files with that word.
  • sudo - run things as a superuser
  • winget - package manager for windows
  • CTRL + c lets you quit / terminate anything that's happening on your terminal
  • wc [filename] - gives you the amount if lines, words, and bytes a file has

editors

  • nano [filename] - opens the file in the terminal with nano! should already be in your terminal
  • notepad [filename] - opens file on notepad

notes

  1. you can use logical operators like && (AND) and || (OR). not really sure in what case i would use OR, but i use && frequently for executing multiple commands at once.
  2. you can hit tab and the terminal will (try to) autocomplete where ever directory or file you want to go.
  3. when typing a file that has an ! you need to escape it using the backslash. if i have a folder called !g/ then you would write \!g/ on the terminal.
  4. files with spaces need to be closed with single or double quotes like this 'folder 1'
  5. use q to quit some scripts
  6. >> this is an append. | this is a pipe
  7. under ~/.profile, ~/.bashrc, or ~/.bash_profile you can set up variables for paths you frequently use. img of my .profile, you can use them in the terminal as variables using this $ symbol. example: cd $DOQMEAT will take me to the path i assigned the variable "DOQMEAT" to.

programs

you have to download this stuff in order to use them!

  • tldr [command] - gives you briefs explanations of what that command does

node.js / npm

  • npx http-server -o [filepath] - local http server for you to preview your site! -o opens the browsers upon executing the command

git

  • git help - help for git commands.
  • git init - initializes a repo
  • git status - list changes
  • git restore [filename] - restores a file
  • git add [files] [dirs] - things to stage for a git commit. you can also use git add -p and it will ask you to yes or no every change before adding it. you can also do git add . to add all changes made to that current directory.
  • git diff [filename] - shows you the changes of that file
  • git commit -m "[your message here!]" - commits changes while also adding a message to them.
  • git push - pushes the code out there.
  • git fetch - fetches code.
  • git pull - pulls code for any changes.
  • git diff --stat --cached [remote/branch] - shows you commits that are about to be pushed. remote/branch is usually origin/main for me.
  • git show - show changes made
  • git log - previous commits
  • rm -rf .git - deletes the .git folder, if you don't want git in your directory then do this.
